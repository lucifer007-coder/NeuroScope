🌟 Getting Started

NeuroScope is an interactive observatory for watching a tiny neural network learn a navigation task, then peering inside it to see why it makes the decisions it does. The task is simple: given an Agent's position and a Target's position on a 2D plane, predict which direction (Up, Down, Left, or Right) the Agent should move to get closer to the Target.

The model is intentionally small — a 4‑input, 32‑neuron MLP — so that a Sparse Autoencoder (SAE) can decompose its internal activity into human-readable "concepts" you can inspect and manipulate in real time.

Recommended first steps

  1. 1Drag the Agent or Target around the Game Arena and watch the live prediction update.
  2. 2Click Train 50 Epochs a few times in the Live Training panel until accuracy climbs above ~95%.
  3. 3Open the Decision Backtrace tab to see the model's reasoning for the current Agent/Target positions.
  4. 4Explore the other tabs — Intervention Sandbox, Feature Observatory, and Game-State Atlas — to go deeper.

🎮 Game Arena

The canvas in the left panel represents a 2D coordinate space from -1 to 1 on both axes.

  • Cyan dot = the Agent. Red dot = the Target.
  • Click and drag either dot (mouse or touch) to reposition it. The network re-runs inference on every move and streams results back over a live WebSocket connection.
  • The arrow drawn from the Agent shows the model's predicted direction; its length and opacity scale with the model's confidence.
  • The numeric coordinates below the canvas update live as you drag.

🧠 Live Training

This panel trains the base MLP and the Sparse Autoencoder together, one batch of synthetic coordinate data at a time.

  • Train 1 Epoch — runs a single training step. Useful for watching metrics move incrementally.
  • Train 50 Epochs — runs 50 steps in one request; a progress bar shows estimated completion.
  • Base Loss / Accuracy / SAE Loss — live metrics for the direction-prediction task and the autoencoder's reconstruction quality.
  • The chart below the metrics plots Base Loss and SAE Sparsity over the full training history.

Training is cumulative — every click continues from the current weights, and progress is checkpointed periodically so it survives a server restart.

🔍 Decision Backtrace

This tab explains the model's current prediction (based on wherever the Agent/Target currently are in the Game Arena).

  • Action Confidence — a bar chart of the softmax probability assigned to each of the four directions.
  • Causal Trace Narrative — a plain-language summary naming the SAE features that most strongly drove the decision.
  • Causal Flow Pathway (Sankey) — a flow diagram tracing Inputs → Neurons → SAE Features → Outputs. Use the Top K slider to control how many neurons/features are shown, or click Show All Neurons for the complete picture.
  • Feature-to-Logit Heatmap — shows, for every one of the 128 latent features, how strongly it pushes the final decision toward each of the four directions.

🧪 Intervention Sandbox

This is where you do causal experiments directly on the network.

  • Each checkbox represents one of the 32 base-MLP neurons. Uncheck one to "prune" it — its activation is forced to zero for every subsequent inference.
  • Reset All restores every neuron; Zero All prunes every neuron at once (useful as a sanity check).
  • The comparison row shows the Original Prediction (with no neurons pruned) side-by-side with the Intervened Prediction (with your current pruning applied).
  • The chart below plots the change in probability for each direction caused by your intervention — bars above zero mean that direction became more likely, below zero means less likely.

Tip: prune one neuron at a time and watch the Decision Backtrace's Sankey diagram to see which causal pathways disappear.

🌌 Feature Observatory

The SAE decomposes the 32 base neurons into a larger dictionary of 128 latent features, most of which are intentionally inactive (sparse) for any given input. This tab lists the features that are currently firing above a small activation threshold, sorted by intensity.

  • Click Zero on any feature card to suppress that feature, similar to pruning a neuron in the Intervention Sandbox — useful for testing whether a specific concept is responsible for a decision.
  • Click Reset Zeroed Features to restore every feature you've zeroed.
  • If you see "No features active above threshold," move the Agent or Target — most coordinate combinations activate at least a few features.

🗺️ Game-State Atlas (Feature Inversion / "Dreaming")

Rather than asking "what does this input activate?", this tab asks the reverse question: "what input would maximally activate this feature?" This is done via gradient ascent directly on the coordinate inputs.

  1. 1Enter a Latent Feature ID between 0 and 127.
  2. 2Click Dream Feature Coordinates.
  3. 3The resulting Agent/Target coordinates that maximize that feature are shown numerically and plotted on the dream canvas.

Note: with 128 features available and only a 4-dimensional input/32-neuron task to explain, many feature slots may never become meaningfully active during training ("dead" features) — dreaming on one of these will simply return the origin (0, 0, 0, 0). Try a handful of different feature IDs to find the ones that have learned an interesting concept; training longer in the Live Training panel tends to bring more features to life.

📦 Bring Your Own Weights

You can train your own version of the base model and/or SAE outside of NeuroScope and load the resulting weights into the live engine.

  • The base model architecture must be exactly CoordinateCatcher (Linear 4→32, ReLU, Linear 32→4).
  • The SAE architecture must be exactly SparseAutoencoder (Linear 32→128, ReLU, Linear 128→32).
  • Export your trained weights with torch.save(model.state_dict(), "my_base_model.pt") (and similarly for the SAE), then upload the .pt file(s) using the pickers or by dragging them onto the dropzone.
  • Files are validated for an exact key/shape match before they're loaded — a mismatched architecture is rejected with a clear error rather than silently corrupting the live model.

⬇️ Exporting Your Work

  • Download Notebook — generates a self-contained Jupyter Notebook with the model code and your current trained weights embedded as base64, so you can keep experimenting offline.
  • Download Report — saves a JSON snapshot of the current Game Arena state, prediction, probabilities, causal narrative, and Sankey diagram data, for use elsewhere or for your own records.

❓ FAQ & Tips

The connection indicator says "Disconnected"

NeuroScope keeps a live WebSocket connection open for real-time inference. If your hosting platform's free-tier instance has spun down, the page will show a "Server is waking up..." overlay and reconnect automatically — this can take 30-60 seconds on the first request.

Why are most SAE features always inactive?

That's expected and intentional — an SAE with more latent features than the underlying task strictly needs is "overcomplete" by design, so it can isolate distinct concepts cleanly. Only a handful of the 128 features will ever fire for this 4-dimensional task.

My intervention isn't changing the prediction

Some neurons and features are redundant or have very little causal weight on the output for the current input — try the Sankey diagram in Decision Backtrace to see which ones actually carry weight to the output before pruning or zeroing them.
